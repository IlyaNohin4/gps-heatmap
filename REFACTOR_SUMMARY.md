## 🎯 Новые правила разработки (CLAUDE.md)

**Процесс:**
1. Задача → читаешь архитектуру → описываешь план
2. Ждёшь подтверждения пользователя
3. Кодишь → коммитишь по запросу
4. Только пользователь может пушить в репо

**Архитектура:**
- Главный файл: `architecture/ARCHITECTURE.md`
- Парсинг деталь: `architecture/PARSER.md`
- Известные проблемы: `POLISH.md`
- Новые решения → сразу обновляешь соответствующий файл

**Безопасность:**
- Никогда не коммитить без OK
- Служебные файлы не в репо (.claude, .env, node_modules, __pycache__)
- Секреты не в чат
- При пуше → проверить что нет .env ключей
