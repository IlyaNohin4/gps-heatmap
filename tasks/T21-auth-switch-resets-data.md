# T21 — Смена аккаунта должна сбрасывать клиентские данные

**Приоритет:** P1 · **Оценка:** 1-1.5h · **Зависимости:** T19 ·
**Происхождение:** баг найден пользователем (2026-07-09)

## Симптом

Logout → login под другим аккаунтом: в сайдбаре остаётся **список треков
предыдущего пользователя** (до перезагрузки страницы). Аналогично под подозрением
список POI и кэш геометрий карты.

## Причина

После T05/T06 списки живут в локальном состоянии компонентов, и их эффекты
не зависят от авторизации:

- `LeftIsland.jsx` — deps эффекта списка: `[buildParams, retryCount, tracksListVersion]`,
  `isAuthenticated` отсутствует;
- `POITab.jsx` — deps списка: только search-подобные, авторизации нет;
- `mapStore.js` — НЕТ reset-действия: `trackDetailCache` (геометрии/детали треков
  прошлого аккаунта) остаётся в памяти; `visibleTrackIds` может содержать чужие id;
- `App.jsx` при смене `isAuthenticated` перезагружает только appStore.tracks,
  geometries и pois — списки компонентов об этом не узнают.

## Что сделать

1. **LeftIsland**: в эффект списка добавь зависимость от `isAuthenticated`
   (из `useAuthStore`). Поведение: если `!isAuthenticated` — очисти
   `items/total/hasMore/error` и НЕ делай запрос (return); при `true` — обычный
   fetch. Гонко-защиту (requestVersion) не ломай.
2. **POITab**: то же самое для списка POI (`listItems/listTotal/listHasMore`).
3. **mapStore**: добавь действие `resetMapData()` — очищает `trackDetailCache`,
   `visibleTrackIds` (и другие принадлежащие-пользователю поля, проверь стор
   целиком; настройки типа activeLayer/theme НЕ трогай). Вызови его из App.jsx
   в ветке `if (!isAuthenticated)` главного data-effect'а (рядом с
   `setTracks([])`).
4. Проверь `appStore`: `selectedTrackId`, `isUploadingIds` — сбрасываются ли
   при logout? Если нет — сбрось там же.
5. Убедись, что при повторном login данные нового пользователя загружаются
   во все три места (список, карта, POI) без перезагрузки страницы.

## Чего НЕ делать

- Не трогать механизм фильтров/пагинации/bump-счётчиков (T19).
- Не сбрасывать пользовательские UI-настройки (тема, язык, слой карты).
- Не менять authStore и логику JWT.

## Критерии приёмки

Сценарий (нужны два аккаунта; второй можно создать на месте):
1. Login аккаунт A → треки A в списке, на карте, POI A.
2. Logout → списки пусты, карта без треков/heatmap прошлого юзера.
3. Login аккаунт B (свежий) → список пуст (у B нет треков), карта пустая,
   POI пусты; загрузка трека под B → появляется у B.
4. Обратно под A → все данные A на месте. Всё — без перезагрузки страницы.

## Как проверить

```bash
docker compose exec -T frontend npm run build
# вручную: сценарий выше (аккаунт B зарегистрировать через UI)
```
