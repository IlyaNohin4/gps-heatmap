# T22 — UI-kit: дизайн-токены и базовые компоненты

**Приоритет:** P1 · **Оценка:** 3-4h · **Зависимости:** нет ·
**Контекст:** пользователь решил (2026-07-10) пересобрать интерфейс «в том же стиле,
но дисциплинированно». T20 (точечные правки отступов) отменена в пользу этого блока.
План блока: T22 (kit) → T23a-e (пересборка островов по одному) → T24 (снос старого).

## Цель

Создать фундамент единообразия: дизайн-токены + 6 базовых компонентов, из которых
будут пересобраны все острова и модалки. Внешний стиль — ТЕКУЩИЙ (те же цвета,
островная компоновка, дух приложения), но с жёсткой геометрией. Плюс демо-страница
для визуальной приёмки пользователем ДО пересборки реальных экранов.

## Что сделать

### 1. Токены — `frontend/src/styles/tokens.css`

Изучи текущие CSS-переменные (`styles/globals.css`, ARCHITECTURE.md § Design System)
— цвета/тени/радиусы уже есть, НЕ дублируй и не переименовывай их. Добавь недостающее:

```css
:root {
  /* spacing scale — единственные допустимые отступы в новых компонентах */
  --space-1: 4px;  --space-2: 8px;  --space-3: 12px;
  --space-4: 16px; --space-5: 24px;
  /* controls */
  --control-h: 32px;        /* единая высота кнопок/инпутов */
  --control-h-sm: 24px;     /* чипы, компактные кнопки */
  /* type scale */
  --text-xs: 11px; --text-sm: 12px; --text-md: 14px; --text-lg: 16px;
}
```

Точные значения выведи из фактического кода (частотный анализ: какие высоты/размеры
сейчас доминируют — те и канонизируй, чтобы «тот же стиль» сохранился).

### 2. Компоненты — `frontend/src/ui/` (новая папка)

Каждый — маленький, с зашитой геометрией, стили через CSS-классы в
`frontend/src/styles/ui.css` (НЕ инлайн):

- **Button** — variants: primary / secondary / ghost / danger; sizes: md (--control-h) / sm;
  флаг iconOnly. Все текущие виды кнопок приложения должны выражаться этими вариантами.
- **Input** — текстовое поле, опционально leftIcon; высота --control-h.
- **Card** — контейнер карточки списка: паддинг --space-3, БЕЗ внешних margin
  (межкарточные отступы — забота контейнера списка через gap).
- **Modal** — оверлей + окно: паддинг --space-5, заголовок, слот действий;
  Escape/клик-мимо — закрытие (сверь с поведением текущих модалок).
- **Panel** — «остров»: существующий стиль обёртки островов (тень/радиус/фон),
  внутренний паддинг --space-3.
- **Chip** — маленькая кнопка-фильтр (как в LeftIsland): высота --control-h-sm.

### 3. Демо-страница — `/ui-demo`

Роут (см. как устроен роутинг в App.jsx), рендерит все компоненты во всех вариантах
и состояниях (hover/disabled/loading где применимо) + шкалу отступов наглядно.
Доступна только в dev-режиме (`import.meta.env.DEV`) — в прод-сборку не попадает.

### 4. Ничего не пересобирать

Существующие острова/модалки НЕ трогать — они переезжают на kit в T23a-e.
Единственное исключение: ничего.

## Чего НЕ делать

- Не менять цвета/тени/радиусы/шрифты — только систематизировать геометрию.
- Не тащить UI-библиотеки (MUI, chakra и т.п.) — свои 6 компонентов.
- Не переписывать существующие компоненты приложения.
- Не трогать сторы, хуки, api.

## Критерии приёмки

- `/ui-demo` в dev показывает все 6 компонентов во всех вариантах; визуально они
  выглядят «как приложение сейчас, но ровно» — финальную приёмку даёт пользователь.
- В прод-сборке роут отсутствует (проверить: `npm run build` + grep по dist на ui-demo).
- В `ui.css` нет значений отступов вне токенов.
- Build зелёный; существующее приложение не изменилось ни на пиксель.

## Как проверить

```bash
docker compose exec -T frontend npm run build
# вручную: http://localhost:5173/ui-demo — приёмка пользователем
# главный экран приложения — без изменений
```
