## Цель

Переписать слой представления LeftIsland (табы Tracks/POI, поиск, фильтры, списки
карточек) на компоненты UI-kit (`frontend/src/ui/`) и токены. Поведение — 1:1.

## Железное правило серии T23

**Данные-слой не трогается.** Хуки, эффекты, состояния, обработчики переносятся
в новый JSX копированием. В них инварианты: debounce+requestVersion (T05/T06),
infinite scroll + «N из M» (T06), tracksListVersion (T19), isAuthenticated-сброс
(T21), error+Retry (T07), display:none-табы (оба смонтированы), фильтры не
влияют на карту.

## Абсолютные запреты (кровью первой попытки)

- ❌ НЕ удалять `poi.css` и любые старые стили — их снос это T24.
- ❌ НЕ править компоненты, объявленные эталоном на текущем чекпоинте.
- ❌ НЕ править файлы вне списка текущего чекпоинта.
- ❌ НЕ делать `git checkout --` без явной команды пользователя.
- Расширение kit — только через вопрос пользователю (согласовано ранее:
  Button получает active prop; регион-теги — span `.track-tag` на токенах,
  НЕ Chip).

## Чекпоинты (после КАЖДОГО: build → скриншот → приёмка пользователем → только потом дальше)

### ЧП-1: TrackCard + контейнер списка треков
Файлы: `TrackCard.jsx`, `LeftIsland.jsx` (только контейнер списка).
Карточка → `Card`: заголовочная строка (бейдж формата + gap --space-2 + название),
строка метаданных, паддинги --space-3 со всех сторон, БЕЗ внешних margin;
межкарточные отступы — gap --space-2 контейнера. Кнопки в карточке → `Button
iconOnly ghost`. Регион-теги в развёрнутой карточке → span `.track-tag`.
**Эталон структуры: как выглядит POICard сейчас (не трогать её!).**

### ЧП-2: шапка вкладки Tracks — поиск, сортировка, фильтры
Файлы: `LeftIsland.jsx` (шапка). Поиск → `Input` с leftIcon; чипы → `Chip`;
кнопка фильтров → `Button` с active prop (добавить в kit + пример на /ui-demo,
это согласованное расширение). Слайдер скорости — оставить как есть, только
отступы на токены.

### ЧП-3: табы и каркас острова
Файлы: `LeftIsland.jsx` (обёртка). Остров → `Panel`, кнопки табов → `Button`
c active. МЕХАНИЗМ переключения (display:none/flex, оба смонтированы) — не
менять ни в коем случае.

### ЧП-4: POI-сторона
Файлы: `POITab.jsx`, `POICard.jsx`. Только теперь POICard можно переводить на
`Card` — приведи к ТОЙ ЖЕ структуре, что принятый в ЧП-1 TrackCard. Кнопки
Import/Create → `Button`. Список — gap контейнера. poi.css НЕ удалять, даже
если классы стали не нужны (заметку — в POLISH.md для T24).

### ЧП-5: финальный прогон
Полная функциональная приёмка пользователем: поиск с debounce, 6 сортировок,
фильтр формата, слайдер, infinite scroll POI с «N из M», мгновенные табы без
запросов, upload → трек в списке, Retry при лежащем backend, смена аккаунта
сбрасывает списки, карта не реагирует на фильтры. Визуально: карточки обоих
списков идентичны по устройству и соответствуют /ui-demo.

## Критерии приёмки

- Все 5 чекпоинтов приняты пользователем ЯВНО (по одному).
- В переписанных файлах нет числовых px-отступов, кроме позиционирования
  острова и динамических цветов — grep-проверка.
- Build зелёный после каждого чекпоинта.
