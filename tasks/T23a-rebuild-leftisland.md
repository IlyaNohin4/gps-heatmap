# T23a — Пересборка LeftIsland на UI-kit

**Приоритет:** P1 · **Оценка:** 2-3h · **Зависимости:** T22 ·
**Блок:** T22 → **T23a** → T23b-e → T24. Первый остров серии — задаёт образец.

## Цель

Переписать НАЧИСТО слой представления LeftIsland (сайдбар: табы Tracks/POI, поиск,
фильтры, список карточек) на компоненты UI-kit (`frontend/src/ui/`) и токены
(`tokens.css`). Поведение и данные — 1:1 как сейчас.

## Железное правило серии T23

**Данные-слой не трогается.** Все хуки, эффекты, состояния, обработчики,
подписки на сторы переносятся в новый JSX КАК ЕСТЬ — копированием, не
переписыванием. В них закодированы выстраданные инварианты:
- debounce 300ms + requestVersion (гонко-защита, T05/T06);
- infinite scroll: useInfiniteScroll + sentinel + счётчик «N из M» (T06);
- tracksListVersion (обновление после upload/delete/rename, T19);
- isAuthenticated в deps + очистка при logout (T21);
- error + Retry (T07);
- переключение табов через display:none/flex — ОБА таба всегда смонтированы (T06);
- фильтры НЕ влияют на карту.

Переписывается только разметка и стили: инлайн-простыни → компоненты kit
(Button, Input, Card, Chip, Panel) + классы ui.css + токены.

## Что сделать

1. Прочитай текущие `LeftIsland.jsx`, `POITab.jsx`, `TrackCard.jsx`, `POICard.jsx`
   и все компоненты kit. Составь соответствие: каждый визуальный элемент → чем
   становится (таблица в плане).
2. Перепиши представление:
   - карточка трека/POI → `Card` (паддинг из kit, БЕЗ внешних margin;
     межкарточные отступы — gap контейнера списка);
   - кнопки/иконки-кнопки → `Button` (variants/sizes/iconOnly);
   - поиск → `Input` с leftIcon;
   - чипы фильтров → `Chip`;
   - обёртка острова → `Panel`;
   - все отступы — только токены --space-*.
3. Инлайн-стили допустимы ТОЛЬКО для позиционирования самого острова
   (fixed/left/top) и динамических значений (цвет формата трека).
4. i18n-ключи, обработчики, пропсы дочерних модалок — без изменений.

## Чего НЕ делать

- Не менять логику (см. железное правило).
- Не трогать модалки (TrackRenameModal и др.) — они в T23e.
- Не менять tokens.css/ui.css под нужды острова без согласования: если kit'у
  не хватает варианта — спроси пользователя, это осознанное расширение kit.

## Критерии приёмки

- Визуально: остров выглядит как на /ui-demo — ровные карточки (симметричные
  паддинги — исходная жалоба пользователя), единые высоты контролов.
  Финальная приёмка — глазами пользователя.
- Функционально, полный прогон: поиск с debounce (1 запрос в Network),
  все 6 сортировок, фильтр формата, слайдер скорости, infinite scroll POI
  (прокрутка с offset-запросами), счётчик «N из M», переключение табов
  мгновенно и без запросов, upload → трек в списке, Retry при лежащем
  backend, смена аккаунта сбрасывает списки. Карта при фильтрации не меняется.
- В новом JSX нет числовых px-отступов (кроме исключений п.3) — grep-проверка.
- Build зелёный.

## Как проверить

```bash
docker compose exec -T frontend npm run build
grep -nE "padding|margin|gap" frontend/src/components/islands/LeftIsland.jsx | grep -vE "var\(--space|// positioning" 
# вручную: полный функциональный прогон из критериев + визуальная приёмка
```
