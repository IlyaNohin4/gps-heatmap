# T26 — Сравнительный аудит статистики: наш пайплайн vs gpx.studio

**Приоритет:** P2 · **Оценка:** 2h · **Зависимости:** T25 (чтобы сравнивать уже
исправленную speed_avg) · **Происхождение:** пользователь хочет перенять методики
расчётов gpx.studio «для всего» (2026-07-10); аудит определит, где это реально нужно.

## Цель

НЕ менять пайплайн. Написать read-only скрипт сравнения: посчитать метрики всех
треков пользователя двумя способами (наш пайплайн — значения уже в БД; методика
gpx.studio — портированная на Python) и выдать таблицу расхождений. По ней
пользователь решит, какие методики перенимать (каждая — отдельной задачей).

## Эталонная методика (проверено по исходникам gpxstudio/gpx.studio, gpx/src/gpx.ts)

1. **Дистанция**: сумма haversine-расстояний соседних точек (по raw-точкам).
2. **Moving-детекция**: пара точек «в движении», если 0.5 ≤ v ≤ 1500 км/ч;
   speed_avg = distance_moving / time_moving (это уже делает T25 — сравнить, что
   наш результат совпадает с портом).
3. **Elevation gain/loss**: RDP-упрощение по высотному профилю (epsilon 20),
   затем скользящее окно ПО ДИСТАНЦИИ 0.1 км, gain/loss по сглаженным значениям.
   (У нас: Савицкий-Голай окном 5 ТОЧЕК — сравнить итоги.)
4. **Уклон**: окно по дистанции 0.05 км.

## Что сделать

1. `backend/scripts/compare_gpxstudio.py` — read-only скрипт:
   - для каждого трека пользователя с raw_points: посчитать по методике
     gpx.studio distance_km, speed_avg (moving), elevation_gain, elevation_loss;
   - взять наши значения из БД;
   - вывести таблицу: track_id, name, метрика, наше, gpx.studio, дельта %.
   - итоговая сводка: по каждой метрике — медианная и максимальная дельта.
2. Запуск: `docker compose exec backend python -m scripts.compare_gpxstudio`.
3. Показать таблицу пользователю. НИКАКИХ изменений пайплайна в этой задаче.
4. По итогам — записать в POLISH.md: какие метрики сошлись (дельта < ~3% —
   считаем эквивалентными), какие разошлись и ждут решения пользователя.

## Чего НЕ делать

- Не менять parser_factory.py, normalizer.py, модели, API.
- Не «чинить» расхождения — только измерить и показать.

## Критерии приёмки

- Скрипт отработал по всем трекам с raw_points без ошибок.
- Таблица + сводка показаны пользователю; выводы записаны в POLISH.md.
