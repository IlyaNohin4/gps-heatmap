# T20 — Выравнивание отступов интерфейса (единая шкала)

**Приоритет:** P2 · **Оценка:** 2h · **Зависимости:** нет ·
**Происхождение:** наблюдение пользователя (2026-07-09): «у карточки отступ снизу
больше, чем сверху, подозреваю, так везде»

## Цель

Привести вертикальные/горизонтальные отступы UI к единой шкале, устранить
асимметрию (разные верх/низ у одного элемента без причины). НЕ редизайн: цвета,
шрифты, структура, размеры компонентов не меняются — только геометрия отступов.

## Что сделать

1. **Инвентаризация.** Собери grep'ом все `padding`/`margin`/`gap` из
   инлайн-стилей компонентов: `components/tracks/TrackCard.jsx`,
   `components/islands/*.jsx`, `components/poi/*.jsx`, модалки, и из CSS-файлов
   (`styles/*.css`). Сведи в таблицу: компонент → верх/низ/лево/право/gap.
   Покажи таблицу мне с пометками, где асимметрия (это твой план — жди OK).
2. **Шкала.** Канонические значения: 4 / 8 / 12 / 16 / 24 px. Каждое значение
   вне шкалы приведи к ближайшему; асимметрию верх/низ устраняй, если для неё
   нет содержательной причины (например, у карточки в списке нижний отступ —
   это межкарточный gap списка, а не padding карточки: такие случаи решай
   через gap контейнера, а не «низ больше верха»).
3. **Карточка трека** (пример пользователя) — разбери первой: паддинги
   карточки должны быть симметричны, расстояние между карточками — gap
   контейнера списка.
4. **Проверка скриншотами.** После правок сними скриншоты (browser preview)
   каждого изменённого блока: список треков, вкладка POI, верх/правый/нижний
   острова, одна модалка. Покажи мне до/после.

## Чего НЕ делать

- Не менять цвета, шрифты, радиусы, тени, размеры иконок.
- Не выносить инлайн-стили в CSS и не рефакторить структуру компонентов
  (возможная отдельная серия задач — обсуждается).
- Не трогать карту и слои.

## Критерии приёмки

- В таблице инвентаризации не осталось значений вне шкалы 4/8/12/16/24
  (осознанные исключения — перечислены с причиной).
- Карточка трека: паддинги симметричны, межкарточное расстояние — gap списка.
- Скриншоты до/после по каждому блоку; финальная приёмка — глазами пользователя.
- `docker compose exec -T frontend npm run build` проходит.
