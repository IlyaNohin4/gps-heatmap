# T18 — Единый стиль уведомлений (i18n + эмодзи)

**Приоритет:** P2 · **Оценка:** 1h · **Зависимости:** T07 · **Происхождение:** пожелание
пользователя при приёмке T07 (2026-07-09)

## Цель

Все toast-уведомления приложения привести к единому стилю: (1) каждый текст — через
i18n (5 языков), без хардкода; (2) эмодзи в начале каждого сообщения для быстрого
визуального различения типов.

## Текущее состояние

- После T07 ошибки загрузки списков локализованы (`errors.tracks_load_failed`,
  `errors.poi_load_failed`, `errors.retry` в `frontend/src/i18n/translations.js`) —
  но без эмодзи.
- Захардкоженные английские тосты (найди ВСЕ grep'ом `toast.` по `frontend/src`),
  известные места:
  - `frontend/src/App.jsx` — `toast.success(\`Track "${file.name}" uploaded\`)`,
    `toast.error(\`Failed to upload "${file.name}"\`)`, аналогичные для POI
    (handleTrackFilesFromOverlay, handlePOIFilesFromOverlay) и другие;
  - `frontend/src/components/upload/UploadZone.jsx` — тосты загрузки;
  - `frontend/src/store/mapStore.js` — `toast.error('Failed to load track geometries')`
    (стор без доступа к React-хукам — см. ниже);
  - компоненты `poi/`, `track/` — проверь grep'ом.
- В `translations.js` уже есть секции `errors`, `tracks`, `poi` — новые ключи клади
  в подходящие существующие секции.

## Что сделать

1. Собери полный список вызовов `toast.*` grep'ом; сведи в таблицу в плане:
   файл, текущий текст, предлагаемый ключ i18n, эмодзи.
2. Конвенция эмодзи (используй её, не изобретай на каждый тост своё):
   - ✅ успех (загрузка/сохранение/удаление прошло);
   - ⚠️ ошибка загрузки данных (то, что можно повторить);
   - ❌ ошибка операции (upload не удался, невалидный файл);
   - 🔄 retry-действия.
3. Для каждого хардкод-тоста: ключ в `translations.js` (все 5 языков: en, es, de, ru,
   uk) с эмодзи в самом тексте перевода, замена вызова на `t('...')` /
   `i18n.t('...')`. Шаблонные значения (имя файла) — через параметры, как принято
   в существующих ключах (`{{shown}} of {{total}}` — образец в `tracks.count_of`).
4. mapStore (не компонент): используй тот же приём, что в T07 —
   `i18n.t(...)` напрямую, если инстанс доступен вне React; иначе оставь английский
   текст, но с эмодзи, и сохрани пометку в POLISH.md.
5. Существующие ключи T07 (`errors.*`) — добавь эмодзи по конвенции.

## Чего НЕ делать

- Не менять логику показа тостов (когда/какие) — только тексты.
- Не трогать бэкенд и тесты.
- Правки translations.js — ТОЛЬКО штатным Edit по якорям соседних ключей,
  не скриптами по номерам строк.

## Критерии приёмки

- `grep -rn "toast\.\(success\|error\|info\|warn\)" frontend/src` — ни одного
  захардкоженного текста (кроме, возможно, mapStore с пометкой в POLISH.md).
- Все тосты начинаются с эмодзи по конвенции; переводы на 5 языках.
- Build проходит; ручная проверка: загрузка трека (✅), битый файл (❌),
  остановленный backend (⚠️ + 🔄 Retry).

## Как проверить

```bash
docker compose exec -T frontend npm run build
grep -rn "toast\." frontend/src --include='*.jsx' --include='*.js' | grep -v "t("
# вручную: загрузить трек, загрузить битый файл, docker compose stop backend → reload
```
